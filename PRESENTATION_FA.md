# راهنمای ارائه پروژه Quoridor (نسخه کوتاه و عملی)

این فایل برای ارائه سریع و روان ساخته شده است تا بدون مکث، پروژه را توضیح بدهی و دمو بگیری.

## 1) اسکریپت 60 ثانیه‌ای شروع ارائه

متن پیشنهادی:

«این پروژه پیاده‌سازی بازی Quoridor در زبان C است.  
پروژه 5 بخش اصلی دارد: نمایش برد ASCII، منطق کامل بازی دو نفره، حالت کاربر-کامپیوتر، ذخیره/بارگذاری باینری، و سیستم Magic Box.  
کد به صورت ماژولار در چند فایل جدا نوشته شده: `main.c` برای حلقه بازی، `game.c` برای قوانین، `io.c` برای ورودی/خروجی، `save.c` برای Save/Load و `ai.c` برای تصمیم‌گیری کامپیوتر.»

## 2) ترتیب پیشنهادی دمو (4 تا 6 دقیقه)

1. نمایش حالت map-file:
```bat
main.exe input.txt
```
بگو: «این قسمت فاز نمایش برد است و ورودی فایل را Parse می‌کند و دیوارها/بازیکن‌ها را دقیق نمایش می‌دهد.»

2. اجرای بازی عادی:
```bat
main.exe
```
- یک بازی PvP یا PvC شروع کن.
- یک حرکت معتبر بزن: `move r c` یا `r c`
- یک دیوار معتبر بزن: `wall r c H`
- یک ذخیره انجام بده: `save demo.qsave`
- بازی را `quit` کن.

3. بارگذاری بازی:
```bat
main.exe
```
در شروع، نام فایل `demo.qsave` را وارد کن.  
بگو: «بارگذاری دقیقا از همان state ادامه می‌دهد (نوبت، دیوارها، موقعیت مهره‌ها، وضعیت‌ها).»

4. (اختیاری) نمایش PvC:
- Mode را `2` انتخاب کن و یک نوبت اجازه بده کامپیوتر بازی کند.
- بگو: «AI تصمیم تصادفی می‌گیرد ولی فقط از بین حرکت/دیوار معتبر انتخاب می‌کند.»

## 3) نقشه ذهنی فایل‌ها برای پاسخ سریع به استاد

- `main.c`
ورود به برنامه، انتخاب mode، حلقه اصلی نوبت‌ها، اجرای commandها.

- `game.c` / `game.h`
قوانین اصلی حرکت، پرش/قطری، دیوارگذاری، بررسی مسیر معتبر با BFS، برد/باخت، Magic Box.

- `io.c` / `io.h`
Parse کردن دستورهای کاربر و چاپ مرتب برد ASCII و وضعیت بازیکن‌ها.

- `save.c` / `save.h`
ذخیره و بارگذاری باینری با هدر/version و اعتبارسنجی state.

- `ai.c` / `ai.h`
ساخت لیست حرکت/دیوار معتبر و انتخاب تصادفی از گزینه‌های معتبر.

## 4) سوال‌های پرتکرار استاد + جواب آماده

1. «چطور جلوی دیوار نامعتبر را می‌گیری؟»
- اول محدودیت‌های هندسی دیوار بررسی می‌شود، بعد با BFS چک می‌شود هر دو بازیکن هنوز مسیر به هدف داشته باشند.

2. «Save چه چیزهایی را نگه می‌دارد؟»
- اندازه برد، mode، نوبت جاری، موقعیت بازیکن‌ها، تعداد دیوارها، آرایه‌های دیوار/بلاک، و اسم بازیکن‌ها.

3. «AI ممکن است حرکت غیرقانونی بزند؟»
- خیر. اول تمام گزینه‌های معتبر تولید می‌شود، بعد از همان‌ها انتخاب تصادفی انجام می‌شود.

4. «Magic Box چه می‌کند؟»
- روی یکی از بازیکن‌ها اثر تصادفی اعمال می‌کند (کم/زیاد شدن دیوار، بلاک شدن نوبت، حذف دیوارها، steal).

## 5) چک‌لیست قبل از ارائه

- `main.exe` اجرا می‌شود.
- `main.exe input.txt` بدون خطا برد را نمایش می‌دهد.
- حداقل یک `save` و یک `load` را همان لحظه دمو کن.
- اگر سوال شد، از روی بخش «نقشه ذهنی فایل‌ها» جواب بده.

## 6) دو خط جمع‌بندی برای پایان ارائه

«پروژه تمام فازهای اصلی تعریف‌شده را پوشش می‌دهد و رفتار بازی در حالت‌های PvP/PvC + Save/Load + Magic Box قابل اجرا و قابل دمو است.  
ساختار ماژولار باعث شده هم نگهداری کد ساده‌تر شود، هم هر بخش مستقل قابل توضیح باشد.»
